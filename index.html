steb version 0.6.6

Use Eclipse as your standard file editor in Linux.

"The files are inside the computer.." is one of my favorite quotes from the movie Zoolander.  However, it's not that easy...there is a problem.   I keep finding myself in the middle of some serious development effort and am coordinating between two or more different editor programs.  Usually it's vi and Eclipse.  Sometimes Mousepad comes along for the ride.  It works alright but causes me extra work, having to keep in mind the key bindings and commands for all these various editors.  Recently I stepped back and said: 'why am I doing this?'  Why not just use one editor and be done with it?  Well, of course Eclipse being my favorite editor the next question was: 'why am I using vi and mousepad all the time?'.  The answer was simple: they are easy to invoke from a shell.  I am an average Linux user and use the shell for many things.  A shell session is like a conversation you have with your machine.  Leaving the shell and going into Eclipse (or any other program) is a mental jump and often it's easier just to run something local to that shell.  So, what I wanted was something that would let me edit a file from the shell as easily as I can with vi our mousepad.  Enter steb.

If you use Linux, Eclipse, and the shell, then this plugin and shell script will make it easier to use Eclipse as a standard file editor.

Features

  • Open and edit files in Eclipse from a shell session.
  • Steb will attempt to open specific editors for known extensions, and will fall back to a standard text editor if there are problems.
  • Create projects from directories with the "-p" option.
  • Use $STEBPORT and Eclipse preferences to allow concurrent sessions.

Screenshot

Usage

After installation, from the shell edit a file with: 

$ steb myfile.txt
Also, create an Eclipse project for a directory with: 
$ steb -p projects/myproject

Dependencies

Plugin: Eclipse 3.5+, Linux Included Client: bash, netcat

Eclipse Updatesite

http://kgilmer.github.com/steb/updatesite

Install

After installing the plugin, copy the client script to somewhere in your PATH. For multiple sessions, set STEBPORT and update the preference page in Eclipse with your session port.

Note, some alternative client scripts are described on this page.

Client Source

#!/bin/bash
# Client script for Linux Shell Eclipse Bridge (steb)
# Set STEBPORT to a specific port.  In shell session you can then
# export this value to allow for multiple shell/eclipse sessions to exist peacefully

if [ -z $STEBPORT ]; then
	STEBPORT=4404
fi

proj_flag=

while getopts 'p' OPTION
do
  case $OPTION in 
  p)	proj_flag=1
                    ;;
  esac 
done

shift $(($OPTIND - 1))

if [ "$proj_flag" ]
then
    echo "-p `realpath $1`" | nc localhost $STEBPORT
else 
    echo "`realpath $1`" | nc localhost $STEBPORT
fi
