steb version 0.6.6

Use Eclipse as your standard file editor in Linux.

If you use Linux, Eclipse, and the shell, then this plugin and shell script will make it easier to use Eclipse as a standard file editor.

Features

  • Open and edit files in Eclipse from a shell session.
  • Steb will attempt to open specific editors for known extensions, and will fall back to a standard text editor if there are problems.
  • Create projects from directories with the "-p" option.
  • Use $STEBPORT and Eclipse preferences to allow concurrent sessions.

Usage

After installation, from the shell edit a file with: 

$ steb myfile.txt
Also, create an Eclipse project for a directory with: 
$ steb -p projects/myproject

Dependencies

Plugin: Eclipse 3.5+, Linux Included Client: bash, netcat

Install

Use http://angelshare.org/org.steb.updatesite within Eclipse update manager to install the plugins. Copy the client script to somewhere in your PATH.

Client Source

#!/bin/bash
# Client script for Linux Shell Eclipse Bridge (steb)
# Set STEBPORT to a specific port.  In shell session you can then
# export this value to allow for multiple shell/eclipse sessions to exist peacefully

if [ -z $STEBPORT ]; then
	STEBPORT=4404
fi

proj_flag=

while getopts 'p' OPTION
do
  case $OPTION in 
  p)	proj_flag=1
                    ;;
  esac 
done

shift $(($OPTIND - 1))
touch $1

if [ "$proj_flag" ]
then
    echo "-p `realpath $1`" | nc localhost $STEBPORT
else 
    echo "`realpath $1`" | nc localhost $STEBPORT
fi
